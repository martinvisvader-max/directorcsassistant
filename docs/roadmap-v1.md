# Roadmap V1

## Objective

Ship a useful internal assistant fast, then improve quality through iteration.

## Phase 1: define the system

Status: `ready now`

Deliverables:

- `MVP spec`
- `architecture`
- `priority taxonomy`
- `escalation definition`
- `output templates`
- `input gap list`

Success condition:

- Martin approves what the assistant should do

## Phase 2: Slack-first prototype

Status: `next`

Deliverables:

- Slack retrieval logic
- normalized storage of source signal
- first-pass classifier for wins, risks, asks, and priorities
- generated daily, midday, and weekly outputs
- Slack DM delivery to `@Martin`

Success condition:

- Martin starts receiving useful briefings from real Slack data

## Phase 3: App V1

Status: `after prototype`

Deliverables:

- private dashboard
- scheduled jobs
- briefing history
- evidence view for source messages

Success condition:

- assistant runs on schedule with stable delivery

## Phase 4: Leadership memory and coaching

Status: `later`

Deliverables:

- 1:1 prep by person
- coaching memory for direct reports
- stronger tone alignment
- first skills matrix layer

Success condition:

- 1:1 prep becomes consistently useful and personalized
